# Refresh System Testing Guide

This guide provides interactive CLI commands to test the refresh system functionality.

## Test Files Created

- `test_data.json` - JSON file for file watching tests
- `test_data.csv` - CSV file for file watching tests
- `test_gen_tasks.sh` - Script that generates dynamic JSON data
- `modify_test_data.sh` - Script that modifies test_data.json

---

## Test 1: Default Behavior (No Refresh)

**Expected:** Refresh is disabled by default, no auto-refresh occurs.

```bash
# View JSON file with default settings (refresh OFF)
python3 -m dv_tui test_data.json
```

While dv is running, modify the file in another terminal:
```bash
./modify_test_data.sh
```

**Expected behavior:** Data does NOT auto-refresh. Press 'q' to exit.

---

## Test 2: Enable Auto-Refresh with Interval

**Expected:** Data refreshes automatically every N seconds.

```bash
# View JSON file with 2-second refresh interval
python3 -m dv_tui --refresh --refresh-interval 2 test_data.json
```

While dv is running, modify the file in another terminal:
```bash
./modify_test_data.sh
```

**Expected behavior:** Data refreshes every 2 seconds, showing updated content.

---

## Test 3: CSV File Refresh

**Expected:** CSV files can also be refreshed.

```bash
# View CSV file with auto-refresh
python3 -m dv_tui --refresh --refresh-interval 3 test_data.csv
```

**Expected behavior:** CSV data refreshes every 3 seconds.

---

## Test 4: Stdin with Command (Dynamic Data Generation)

**Expected:** Data refreshes by re-running the specified command.

```bash
# View data generated by command, refresh every 2 seconds
python3 -m dv_tui -c "./test_gen_tasks.sh" --refresh --refresh-interval 2
```

**Expected behavior:** Data changes every 2 seconds with different timestamps and item counts.

---

## Test 5: Stdin Without Command (Cannot Refresh)

**Expected:** Cannot refresh piped data without command.

```bash
# Pipe static JSON to dv
cat test_data.json | python3 -m dv_tui
```

Press 'r' (if you implement manual refresh key) or wait for auto-refresh.

**Expected behavior:** Shows warning message "Cannot refresh piped data without command".

## Test 6: Inline Config Refresh Settings

**Expected:** Refresh settings in `_config` field are respected.

```bash
# View file with inline refresh config
python3 -m dv_tui test_data.json
```

The test_data.json contains:
```json
{
  "_config": {
    "refresh": {
      "enabled": false,
      "interval": 2.0
    }
  }
}
```

**Expected behavior:** Refresh is disabled despite inline config. (CLI flags override inline config)

---

## Test 7: Preserve Selection and Scroll on Refresh

**Expected:** When data refreshes, your current selection and scroll position are preserved.

```bash
# View data with auto-refresh
python3 -m dv_tui --refresh --refresh-interval 2 test_data.json
```

1. Navigate to a specific row (e.g., row 3)
2. Wait for refresh to occur
3. Modify file: `./modify_test_data.sh`

**Expected behavior:** After refresh, you should still be on row 3 (or nearest available if row count changed).

## Test 8: On-Trigger Refresh (with configured triggers)

Create a test file with triggers:

```bash
cat > test_triggers.json << 'EOF'
[
  {
    "_config": {
      "refresh": {
        "enabled": false,
        "on_trigger": true
      },
      "triggers": {
        "table": {
          "on_select": {
            "command": "echo 'Selected:' > /tmp/dv_trigger.log",
            "async": false
          }
        }
      }
    },
    "type": "work",
    "status": "active",
    "summary": "Task 1"
  }
]
EOF

# View with on-trigger refresh
python3 -m dv_tui test_triggers.json
```

**Expected behavior:** When you select a row (press Enter), the trigger executes and data refreshes.

## Manual Refresh Testing

If you want to test manual refresh (pressing a key to refresh), you can use the `refresh_data` method directly:

```bash
python3 << 'EOF'
from dv_tui.core import TUI
from dv_tui.config import load_config

# Create TUI instance
config = load_config()
tui = TUI(['test_data.json'], config=config)

# Test refresh functionality
tui.load_data()
print(f"Loaded {len(tui.current_tab.data)} items")

# Simulate file modification
import os
os.system('./modify_test_data.sh')

# Refresh
success = tui.refresh_data(show_warning=False)
print(f"Refresh {'succeeded' if success else 'failed'}")
print(f"New data: {len(tui.current_tab.data)} items")
EOF
```

## Summary of Flags

| Flag | Description | Default |
|------|-------------|----------|
| `--refresh` | Enable auto-refresh | OFF |
| `--refresh-interval N` | Set refresh interval to N seconds | 1.0 |
| `-c COMMAND` | Use COMMAND to generate data (required for stdin refresh) | None |

---

## Troubleshooting

**Refresh not working?**
- Check that `--refresh` flag is present
- Verify `--refresh-interval` is a positive number
- Ensure command exists and is executable (for `-c` usage)

**Data not updating after file changes?**
- File watching works via modification time (mtime)
- Make sure you're actually modifying the file, not just viewing it
- Test with `./modify_test_data.sh` to ensure proper modification

**Stale data displayed?**
- Press `q` to exit and restart dv
- Check that the data source is actually generating new data
- For `-c` usage, test the command manually: `./test_gen_tasks.sh`
